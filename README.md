# 灵工保：劳务报酬代开 + 以票扣保 Demo

这是一个同时提供 **Next.js 版本** 与 **静态 HTML 版本** 的灵工保演示项目，核心场景为：

- 企业发起劳务报酬代开申请
- 个人在移动端确认
- 模拟税费处理并生成数电发票
- 按开票金额 1% 联动沉淀社保储备金
- 同步更新社保健康度、账户余额与 AI 顾问答复

## 当前已完成的功能

### 1. 劳务结算中心
- 首页新增“待办劳务费结算”卡片
- 展示 `某某技术公司` 劳务费 `¥2,000.00`
- 展示预计税费 `¥20.00`
- 提供“确认开票并预扣社保”主操作按钮

### 2. 开票 + 社保联动逻辑
- 点击按钮后弹出“政务端跳转确认”弹窗
- 确认后执行 `开票金额 * 1%` 的社保储备金模拟计算
- 实时更新：
  - 账户余额
  - 社保健康度
- 成功后展示提示语：
  - `发票开具成功！已同步为您存入 20 元社保储备金，保障不掉线。`

### 3. AI 顾问知识库更新
当用户输入以下关键词时：
- `发票`
- `领工资`
- `报酬`

AI 顾问会回复：

> 根据 2026 最新政策，您只需在灵工保确认企业发起的代开申请即可一站式领薪缴保。

## 代码结构

```text
app/
  globals.css        Next.js 全局样式
  layout.tsx         Next.js 元数据与根布局
  page.tsx           Next.js 首页主界面
app.html             静态产品页
index.html           静态入口页
style.css            静态页面样式
server.js            本地静态预览服务
deploy-static/       已同步好的静态部署目录
```

## 本地预览

### 方式一：静态演示版（默认）
```bash
npm run dev
```

然后访问：
- `http://localhost:3000`

当前 `server.js` 默认从 `deploy-static/` 提供静态页面。

### 方式二：检查 Next.js 代码
如需继续修改 Next.js 页面，重点查看：
- `app/page.tsx`
- `app/globals.css`
- `app/layout.tsx`

## 部署说明

静态部署推荐直接使用：

- `index.html`
